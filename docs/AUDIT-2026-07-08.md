# Audit — 2026-07-08

A full read of the engine (this repo) and of instance #1 (`../fundaimentals`, the live
AI-engineering course), looking for improvement vectors to take the harness past its
prototype state. Each item below is written to be actionable without this audit's
conversation: problem → evidence → proposed mechanism → files involved.

**Central finding:** the architecture is validated (engine/instance path disjointness holds —
shared files verified byte-identical between the two repos; pedagogy formats are porting
well), and the engineering is careful where it matters (CSP-sandboxed visuals, path-escape
guards, DOMPurify, fail-safe PTY classification, graceful JSON degradation). The weakness is
uniform: **the protocol describes discipline but nothing enforces it.** Every failure that has
occurred in production is an enforcement failure, not a design failure. The theme of this
backlog is converting earned prose rules into mechanisms — the repo's founding ethos applied
to itself.

## Live evidence: the enforcement gap has already failed twice

1. **A protocol edit was silently lost for ~2 weeks** (recorded in the instance's
   `GENERALIZATION.md` and journal): four quiz-protocol rules codified in the 2026-06-16
   session were found missing in the 2026-07-02 audit and had to be re-added.
2. **As of this audit, the instance sits in a desynced, uncommitted state:**
   `tutor/quiz-bank.json` has uncommitted grading from a 2026-07-05 session (6+ items graded
   after the ~3-week break), but `tutor/journal.md` has **no 2026-07-05 entry** (last entry
   2026-07-03), `tutor/progress.json` was untouched (mtime 06-16), and nothing was committed
   (last commit 07-04). This is precisely the failure the "commit at session close" rule was
   written to prevent. *(Instance-side fix: next fundaimentals session must reconcile —
   journal the 07-05 session retroactively, sync progress.json, commit. Engine-side fix:
   Tier 1 below.)*

Other production incidents feeding this backlog: sealed-reference QA repeatedly bypassed by
materials bugs the **learner** caught (module 00 price-key mismatch; hardcoded dollar values;
a flaky `secondMs < firstMs/2` timing check; a lab opening on the wrong config; a module-04
refusal-contract hole that would have graded correct refusals as failures). The 07-02
full-repo audit fixed ~15 issues — QA-by-discipline is expensive and leaky.

## Tier 1 — State integrity: make the protocol self-enforcing (highest leverage)

All items preserve the invariant that the study owns zero state — these are **verifiers**,
never writers.

### 1.1 Session-close doctor
- **Problem:** session close (update progress + quiz-bank + journal + commit) is prose; it
  half-completed on 2026-07-05 in production.
- **Mechanism:** `scripts/doctor.mjs` (`npm run doctor`) detecting desync:
  `tutor/quiz-bank.json` modified without a same-day `journal.md` entry; dirty git tree older
  than N hours; `progress.json.currentModule` inconsistent with the journal's last entry.
  The study surfaces failures as a topbar banner ("⚠ unclosed session from Jul 5") with a
  quick action that types a reconciliation opener into the agent (pattern exists:
  `study/src/components/TerminalPane.tsx` quick actions + `study/server/index.ts` PTY state).
  Add one line to `CLAUDE.md`'s session protocol: run the doctor at open and close.

### 1.2 Atomic close checklist in the protocol
- **Problem:** `CLAUDE.md` session-close step lists updates as narrative.
- **Mechanism:** restate as an explicit all-or-nothing checklist (progress + quiz-bank +
  journal + commit), with the doctor (1.1) as its mechanical check.

### 1.3 JSON Schemas + `npm run validate`
- **Problem:** format vocabulary is drifting in the wild: `"completed"` (disk,
  `docs/FORMAT.md`, templates) vs `"complete"` (UI) — papered over by a normalize in
  `study/server/index.ts` (~line 71); `volatileLayer: "present"` (module 00) vs
  `"generated-at-start"` (01–04); `bossCheck` present on only one manifest; free-form status
  strings feed unvalidated `status-${…}` CSS class names in `Rail.tsx`/`DocPane.tsx`.
- **Mechanism:** JSON Schemas for `module.json`, `progress.json`, `quiz-bank.json`,
  `lab.json` under `docs/schema/` (or `schemas/`); a validate script run in CI and available
  to the tutor after generation. Pick one status vocabulary and migrate.

### 1.4 Deterministic spaced-repetition arithmetic
- **Problem:** the interval math (correct ×2.5 min-2-days / partial→2 / wrong→1, due dates)
  is done by the LLM by hand. Production bug (06-16): close-time seeding flattened earned
  intervals; module 01's quiz silently never ran. The synthetic `"rescheduled"` history
  result exists only to keep bookkeeping from faking grades — format strain.
- **Mechanism:** `scripts/quiz.mjs` — `quiz.mjs due` (session-open pick, most-overdue first),
  `quiz.mjs grade <id> correct|partial|wrong [note]` (arithmetic + history append). The tutor
  still judges; the script does bookkeeping. Update `CLAUDE.md` session protocol to route
  through it.

## Tier 2 — Guard the engine/instance boundary and updates

### 2.1 Boundary check
- **Problem:** nothing mechanically prevents an instance editing an engine file; one stray
  edit breaks `git pull` updates permanently. The rule lives only in `CLAUDE.md` prose.
- **Mechanism:** before pulling, `scripts/update.mjs` diffs engine paths against the remote;
  on divergence, refuse loudly with recovery instructions. Optionally seed a pre-commit hook
  at onboarding.

### 2.2 Harden `scripts/update.mjs`
- **Problem:** bare `git pull` with no dirty-tree check; `npm install` after a dependency
  bump is a printed reminder only.
- **Mechanism:** refuse (or auto-stash) on dirty tree; detect `package-lock.json` change
  post-pull and run `npm install` automatically.

### 2.3 Bootstrap installer (already on `docs/ROADMAP.md`)
- **Problem:** correct setup (named clone; `upstream`=engine, `origin`=backup) is a manual
  dance; GitHub's "Use this template" severs history and breaks updates (README warns, but
  warnings aren't mechanisms).
- **Mechanism:** `npx`-style bootstrap: clone named after the course, wire remotes, install,
  open the study.

## Tier 3 — QA automation for generated materials

### 3.1 `scripts/qa-module.mjs` — mechanize sealed-reference QA
- **Problem:** the QA ritual (reference solution green → scaffold fails on assertions →
  delete reference) is pure discipline and has leaked bugs to the learner repeatedly.
- **Mechanism:** a harness that runs the module's checks against a reference, then the
  scaffold, asserting the green→red-on-assertions pattern; verifies `hints/hint-{1,2,3}.md`
  exist; `quiz.md` has 4–8 questions; `lab.json` references valid stock keys
  (`study/src/lab/registry.ts`) or existing `visuals/*.html`; visual HTML contains no
  external URLs (lint what the serve-time CSP will block).

### 3.2 Heuristic materials linter
- Scaffold contains `// TODO(you):` gaps; hint-2 contains no pasteable code fences (the
  "demote to hint-3" rule from `CLAUDE.md`); checks contain no relative timing assertions
  (the known flake anti-pattern).

### 3.3 Scaffold hygiene
- **Problem:** instance #1 has `node_modules/` committed under every module scaffold.
- **Mechanism:** `.gitignore` guidance in `docs/FORMAT.md` + a QA-script flag.

## Tier 4 — Study app: from lens to instrument panel (all read-only)

- **4.1 Quiz visibility:** "due today: N" topbar chip; a retention view (items, intervals,
  history). `tutor/quiz-bank.json` is currently not surfaced anywhere in the UI. Also makes
  desync (1.1) visible to the learner.
- **4.2 Journal reader:** `tutor/journal.md` is the richest artifact in the system (30 KB of
  real pedagogy in instance #1) and is invisible. Typeset, newest-first.
- **4.3 Progress analytics:** hints used, check attempts, time-per-module, phase completion —
  already in `progress.json`, just unrendered.
- **4.4 Check-run status in the rail:** parse the last "run checks" output (ephemeral or a
  gitignored cache) so red→green is visible where progress is displayed.
- **4.5 Light mode + accessibility:** dark-only, no `prefers-color-scheme`; lab overlay lacks
  a focus trap; the draggable vector lab is pointer-only; status conveyed by glyph/color
  alone. Rename leftover `ck.*` localStorage keys (`App.tsx`: `ck.railW`, `ck.termW`,
  `ck.selected`) from the cockpit era.
